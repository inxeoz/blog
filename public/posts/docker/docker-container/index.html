A container is a running instance of a Docker image ‚Äî a lightweight, isolated environment that can hold your software, tools, and files.

  • Container = the living instance of that image

When you stop or delete a container, your data and changes can vanish ‚Äî unless you know how to make them persistent.

This creates a new container with all your changes already baked in ‚Äî like a snapshot.
